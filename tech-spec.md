Technical Specification: MJML Template Generator (v2.0)1. System ArchitectureThe application follows a Unidirectional Data Flow:JSON State (Zustand) → MJML Transformer → MJML-to-HTML Compiler → Iframe Preview.2. Core Data Models (TypeScript Interfaces)These definitions are the "source of truth" for the LLM to generate components.3. Custom Blocks Definition (Composite Schema)"Blocks" are not new MJML tags; they are pre-configured JSON trees. When a user selects a block, the application pushes a predefined MJElement structure into the body.3.1 Grid (Multi-column)Logic: A mj-section containing $N$ number of mj-columns.Initial State: Two mj-columns each containing a mj-spacer (placeholder).3.2 CardStructure:mj-section (Attributes: background-color, border)mj-columnChildren: mj-image (Top) -> mj-text (Title/Body) -> mj-button (CTA).3.3 Email SignatureStructure: Uses mj-group inside a section to ensure the image and text stay side-by-side on mobile.mj-column (Width 25%): mj-image (Profile/Logo).mj-column (Width 75%): mj-text (Name/Contact info).4. The Engine: JSON-to-MJML TransformerThis is the most critical logic for an LLM to understand. It is a recursive function that converts our JSON tree into a valid MJML string.5. Feature Specifications5.1 The Drag-and-Drop / Addition LogicConstraint: Atomic components (mj-text, etc.) cannot be direct children of a mj-section. They must be inside a mj-column.Automation: When a user adds a "Text" component to an empty section, the system must automatically wrap it in an mj-column.5.2 The Property Editor (Right Sidebar)The editor must be context-aware.If activeElement.type === 'mj-text', show: fontSize, fontWeight, color, textAlign.If activeElement.type === 'mj-image', show: src, alt, width, href.5.3 Live Preview ImplementationTo prevent lag, the preview should use:Debounce: 300ms delay after state change before re-compiling.mjml-browser: Client-side compilation to avoid network latency.Iframe srcDoc: Injects the compiled HTML safely.6. Implementation Roadmap (Iterative)Milestone 1: State Management (Zustand) and the jsonToMjml transformer.Milestone 2: Canvas rendering with basic "Add Component" functionality.Milestone 3: Attribute Editor (Right Sidebar) with two-way data binding.Milestone 4: "Ready-made" Block library and MJML Export.
